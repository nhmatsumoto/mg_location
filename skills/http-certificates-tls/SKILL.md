# HTTP + Certificados + TLS

## Quando usar
Para definir padrões de comunicação HTTP e segurança em trânsito.

## Checklist
1. Aplicar status codes e headers corretos.
2. Definir timeout/retry/circuit-breaker para clientes HTTP.
3. Validar cadeia de certificados e expiração.
4. Forçar HTTPS em ambientes expostos.
5. Documentar políticas de cache e idempotência.
