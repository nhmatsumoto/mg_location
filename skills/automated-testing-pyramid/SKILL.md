# Testes Automatizados (Unidade + Integração)

## Quando usar
Para ampliar confiabilidade de mudanças e reduzir regressões.

## Checklist
1. Priorizar testes de unidade para regras de negócio.
2. Cobrir integrações entre módulos e APIs críticas.
3. Definir massa de dados de teste estável.
4. Automatizar execução em pipeline local/CI.
5. Garantir assertions de erro, borda e caminho feliz.
