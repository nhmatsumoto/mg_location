# Design Patterns na Arquitetura

## Quando usar
Para resolver variação de comportamento/integração sem aumentar acoplamento.

## Checklist
1. Mapear problema e pontos de variação.
2. Selecionar padrão adequado (Strategy, Factory, Adapter, Observer etc.).
3. Explicitar trade-offs e custo de complexidade.
4. Validar com testes de unidade focados no comportamento.
5. Evitar overengineering quando solução simples for suficiente.
