# Web + React + TypeScript + Zustand

## Quando usar
Para evolução de frontend com componentização tipada e estado global previsível.

## Checklist
1. Quebrar telas em componentes pequenos e reutilizáveis.
2. Tipar props, estados e contratos com API.
3. Centralizar estado compartilhado no Zustand com selectors.
4. Isolar integração REST em serviços dedicados.
5. Garantir feedback de loading/erro e UX acessível.
