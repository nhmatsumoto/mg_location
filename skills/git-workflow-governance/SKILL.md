# Git Workflow & Governança de Mudanças

## Quando usar
Para padronizar fluxo de versionamento, revisão de código e entrega contínua.

## Checklist
1. Definir convenção de branch e commits semânticos.
2. Garantir PR com contexto, risco, testes e rollback.
3. Aplicar estratégia para hotfix/release branch.
4. Evitar merge sem validação técnica mínima.
5. Registrar decisões de versionamento no learning log.
